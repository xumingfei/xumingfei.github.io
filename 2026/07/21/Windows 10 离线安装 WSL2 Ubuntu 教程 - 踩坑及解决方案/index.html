Windows 10 离线安装 WSL2 Ubuntu 教程 - 踩坑及解决方案

发表于2026-07-21|更新于2026-07-21|环境搭建
|浏览量:

近期需要在 Windows 中安装 Docker，前置步骤需要 WSL2 + Ubuntu。本文整理完整的离线安装过程以及踩过的坑，供大家参考。

一、准备工作

1. 系统要求

  • 操作系统：Windows 10 版本 2004 及更高（Build 19041+）或 Windows 11
  • 硬件虚拟化：在 BIOS/UEFI 中启用 Intel VT-x / AMD-V
  • 存储空间：至少 5GB 空闲空间（建议 SSD）

必须运行 Windows 10 2004+ 或 Windows 11 才能使用 wsl --install 命令。早期版本请参考官方手动安装页。
如果直接执行 wsl --install -d Ubuntu-22.04 速度非常慢，就需要采用下文的离线安装方式。

2. 启用 WSL 相关功能

以管理员身份打开 PowerShell，执行：

3. 通过图形界面手动启用（可选）

也可以走控制面板路径：

  1. 打开【控制面板】→【程序】→【启用或关闭 Windows 功能】
  2. 勾选 虚拟机平台、适用于 Linux 的 Windows 子系统（如有 Hyper-V，建议一并勾选）
  3. 点击确定后重启计算机

4. 确认 BIOS 已启用虚拟化

进入 BIOS 设置，确保虚拟化技术（Intel VT-x 或 AMD-V）已启用，否则后续启动 WSL 会报错。

二、手动下载 Ubuntu WSL 镜像

1. 获取官方镜像

  • 官方下载地址：Ubuntu 22.04.5 LTS - Microsoft Store
  • WSL 官方文档：Windows Subsystem for Linux Documentation

⚠️ 踩坑提示：文档页面中找到 Manual install steps 链接后，直接点击很可能返回”无访问权限”。实测从 Microsoft Store 的下载入口直接拉取 .AppxBundle 文件是最稳妥的方式。

2. 备用镜像源

如果官方源下载慢，可以使用清华源镜像站：

  • 清华大学开源软件镜像站：https://mirrors.tuna.tsinghua.edu.cn

3. 保存镜像文件

将下载得到的 Ubuntu2204-221101.AppxBundle 保存到本地固定目录，例如 C:\wsl\。

三、离线安装 Ubuntu

1. 以管理员权限打开 PowerShell

【Windows PowerShell（管理员）】或【Windows Terminal（管理员）】均可。

2. 进入镜像所在目录

3. 执行安装命令

⚠️ 注意事项

  1. 必须使用管理员权限打开 PowerShell，否则会遇到权限不足的报错。
  2. 确认文件名准确，可以先用 ls 或 dir 命令查看下载目录中的文件。
  3. 文件名输入时可用 Tab 键自动补全。
  4. 如果安装失败，可以尝试旁加载（Sideload）：
    • 打开【Windows 设置】→【应用】→【高级应用设置】
    • 找到 开发人员模式 或 从任意源安装应用 选项并启用
    • 重新执行 Add-AppxPackage

四、验证安装是否成功

开始菜单中出现 Ubuntu 的图标即表示安装成功：

至此，Ubuntu 22.04 已成功安装。接下来启动它时会进行 WSL 首次初始化，要求创建一个默认的 UNIX 用户账户，用于后续日常操作。

五、启动 Ubuntu 并初始化用户

1. 设置 UNIX 用户名

启动后按提示输入用户名，注意：

  • 不能使用大写字母
  • 不能包含空格
  • 该用户名将成为默认用户，并拥有 sudo 权限（类似 Windows 的管理员权限）

2. 设置密码

输入密码时屏幕不会显示任何字符，这是 Linux 的安全特性，按 Enter 确认即可。

密码建议：

  • 务必牢记，后续使用 sudo 时需要输入
  • 至少 6 个字符
  • 建议使用大小写字母、数字、特殊字符的组合

六、常见问题速查

问题 原因 解决方案
wsl --install 卡住或超慢 官方源网络问题 采用本文离线安装方式
安装报权限不足 未使用管理员权限 以管理员身份运行 PowerShell
启动报 WslRegisterDistribution 错误 BIOS 未开启虚拟化 进入 BIOS 启用 VT-x / AMD-V
Add-AppxPackage 失败 系统未允许旁加载 启用开发人员模式或旁加载应用
WSL 版本仍为 1 未设置默认版本 执行 wsl --set-default-version 2
