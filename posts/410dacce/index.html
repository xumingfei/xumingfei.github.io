一、核心原理

SSH 密钥配对，本地生成公钥上传 GitHub，后续 git pull / git push 不用输账号密码，HTTPS 地址换成 SSH 地址使用。

三、2. GitHub 添加公钥

  1. 网页登录 GitHub → 右上角头像 → Settings
  2. 左侧菜单：SSH and GPG keys → New SSH key
  3. Title 随便填（例如 PC-本机）
  4. Key 粘贴刚才复制的全部公钥内容，保存，验证一次 GitHub 密码。

常见报错解决

    • 公钥没复制完整、没上传到 GitHub；
    • 生成密钥时路径不对，确认 ~/.ssh/ 文件存在。
    • SSH 仓库地址抄写错误；用户名 / 仓库名大小写不对。

  3. 推送依然要密码

    • 远程地址没成功切换成 git@github.com 开头的 SSH 链接。

  4. Windows 文件夹权限归属和当前登录用户不一致，Git 判定目录存在安全风险（可疑仓库所有权），阻止你操作 remote。按提示执行一行命令放行即可。

git remote add origin git@github.com:xxx/hexo-blog-source.git
